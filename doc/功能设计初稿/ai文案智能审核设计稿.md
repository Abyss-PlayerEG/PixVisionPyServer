# AI文案内容审核功能接口设计文档

## 1. 接口概述

对用户输入的文案进行自动化内容审核，输出三种状态：**违规（violation）**、**中立（neutral）**、**正常（normal）**。  
设计目标：输入精简，响应快速，输出结构化JSON，便于后端解析及Agent集成。

---

## 2. 接口定义

### 2.1 基本信息

- **路径**：`POST /v1/content/audit`
- **请求格式**：`application/json`
- **响应格式**：`application/json`

### 2.2 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `text` | string | 是 | 待审核的文案内容，长度限制 1-5000 字符（可根据模型调整） |

**请求示例**：
```json
{
  "text": "今天天气真好，一起去公园玩吧。"
}
```

### 2.3 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| `status` | string | 审核结果：`normal` / `neutral` / `violation` |
| `reason` | string | 判断依据简述（30字以内，便于调试） |
| `confidence` | number | 可选，置信度 0-1（视模型能力提供） |

**响应示例**：
```json
{
  "status": "normal",
  "reason": "内容为日常积极描述，无违规迹象",
  "confidence": 0.98
}
```

---

## 3. 状态定义

| 状态 | 英文标识 | 含义 |
|------|----------|------|
| 正常 | `normal` | 内容符合法律法规及公序良俗，无任何风险 |
| 中立 | `neutral` | 无明显违规，但存在擦边、争议观点、广告软文、轻微敏感词或需人工进一步判断 |
| 违规 | `violation` | 明确包含色情、暴力、仇恨言论、政治敏感（如分裂国家、攻击领导人）、诈骗、违禁品交易、侮辱诽谤、侵权等违法或严重不良信息 |

---

## 4. AI提示词设计

### 4.1 系统提示词（固定）

```text
你是一个内容安全审核助手。你的任务是对用户输入的文案进行分类，判断其属于“正常”、“中立”还是“违规”。

定义标准：
- 正常（normal）：内容符合法律法规及公序良俗，无任何风险。
- 中立（neutral）：内容无明显违规，但包含擦边、争议观点、广告软文、轻微敏感词汇、可能引起不适但不违法，或需要人工进一步判断。
- 违规（violation）：明确包含色情、暴力、仇恨言论、政治敏感（如分裂国家、攻击领导人）、诈骗、违禁品交易、侮辱诽谤、侵权等违法或严重不良信息。

你必须严格按照以下JSON格式返回结果，不要输出任何其他解释、Markdown或多余文本：
{"status": "normal|neutral|violation", "reason": "简短中文理由"}

其中 reason 长度不超过30字。
```

### 4.2 用户提示词（动态填充）

```text
请审核以下文案：
{{文案内容}}
```

> 调用时将 `{{文案内容}}` 替换为请求中的 `text` 字段。

---

## 5. 设计思路

### 5.1 核心原则
- **快速响应**：单次审核耗时控制在2秒内（取决于模型）。
- **可解释性**：返回简短原因，便于人工复核。
- **安全边界**：提示词明确禁止模型输出主观评价或扩展内容，仅按指定格式返回JSON。
- **状态简化**：仅三种状态，覆盖绝大多数审核场景。

### 5.2 技术架构建议
- 输入 → 前置过滤（可选，如正则命中极端敏感词直接返回`violation`）→ 调用LLM → 解析JSON → 返回结果。
- 可增加缓存机制：相同文案短时间内重复请求，直接返回缓存结果。
- 降级策略：若LLM超时或返回非JSON，默认返回`neutral`并记录异常。

### 5.3 扩展能力（可选）
- 多语言支持：提示词中增加自动语言识别，reason使用原文语言输出。
- 批量审核：接口支持数组`texts`，每条分别返回结果。
- 自定义敏感词库：结合AI与本地词库，提高效率和准确性。

## 6. 调用示例

### 6.1 正常内容
**请求**：
```json
{"text": "今天天气真好，一起去公园玩吧。"}
```
**LLM输出**：
```json
{"status": "normal", "reason": "日常积极描述，无违规"}
```
**接口响应**：
```json
{"status": "normal", "reason": "日常积极描述，无违规", "confidence": 0.99}
```

### 6.2 中立内容
**请求**：
```json
{"text": "这个产品虽然有人说不好，但我觉得还行，你可以试试。"}
```
**LLM输出**：
```json
{"status": "neutral", "reason": "含轻度广告诱导，无明确违规"}
```
**接口响应**：
```json
{"status": "neutral", "reason": "含轻度广告诱导，无明确违规", "confidence": 0.82}
```

### 6.3 违规内容
**请求**：
```json
{"text": "点击链接免费领取1000元红包，名额有限！"}
```
**LLM输出**：
```json
{"status": "violation", "reason": "疑似诈骗或诱导点击"}
```
**接口响应**：
```json
{"status": "violation", "reason": "疑似诈骗或诱导点击", "confidence": 0.95}
```

---

## 7. 错误处理

| 场景 | 返回状态 | 说明 |
|------|----------|------|
| 输入`text`为空或超过长度限制 | HTTP 400 | 返回错误信息，不调用LLM |
| LLM调用超时（>3秒） | `neutral` | 降级返回中立，记录日志 |
| LLM返回非JSON格式 | `neutral` | 降级返回中立，记录原始输出 |
| LLM返回的status不在枚举中 | `neutral` | 容错处理 |
